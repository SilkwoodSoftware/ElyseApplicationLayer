# User-Related Lists

Users can be directly linked to the following.

- Documents

- Files

- Duty functions

- People Lists

- Personal [tag browsing trees](tag-management.md)

- [Workflow instance steps](workflow-management.md)

Controllers can also be linked to [controller file groups and controller document groups](controller-groups.md) to provide metadata editing rights which are exclusive within the group.  Controllers can also be linked to [form groups](forms.md) as a form group owner to provide form editing rights which are exclusive within the group. 

**Duty Functions** are intended for organizational duty functions, for example `Safety Manager`.  Duty functions in turn can be grouped into **Function Lists**.   

Function lists can be linked to the following.

- Documents

- Files

- [Document groups](document-groups.md) for metadata edit permissions

- [File Groups](file-group-editor-permissions.md) for metadata edit permissions

- Tag Browsing Tree permissions

- Tag Group permissions

- Tag Relationship Tree permissions

**People Lists** are provided as a means of directly grouping users without needing to assign duty functions. 

User-related fields can be linked to a **General Field Name**, for example `Author`.  The following can be linked to a general field name.

- Document to Duty Function links

- Document to Function List links

- Document to People List links

- Document to User links

- File to Duty Function links

- File to Function List links

- File to People List links

- File to User links
